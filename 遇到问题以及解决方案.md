# 一、RAG部分
## 1、数据清理
### 文档格式转换
    - docx, txt格式文档：采用markitdown转换成markdown文件
    - pdf格式文档：采用docling框架转换成markdown文件（基于OCR识别的相关模型）
    - excel格式文档：采用pandas转换成md文件
    - 格式转换：采用离线处理。
### markdown文档处理成chunk
    - 将全角字符改成半角字符（跨平台兼容好）、删除连续的空行、清除html的相关标记（有的文档是由html转换过来的）、中文标点转换成英文标点（便于统一处理）。
    - 按{#， ##， ###， ####}四级标题切分，元数据记录标题路径 header1 > header2，方便后续进行查找对应内容。
    - 递归分割，先按设置的字符列表顺序进行递归切割（先按第一个字符切割，如果还有超过chunk_size的块再按第2个字符切割，最后相邻的块添加前一个块的一部分内容，保持连贯性）。
    - 删除chunk可以通过vectore.get_pk()获取每个documents的主键（相当于代码种设置的ids和表中的milvus_id字段），检索出的文档的metadata.pk就是ids。


# 二、智能体部分
## 1、上下文管理
### 上下文组成（模型上下文长度为m）
- 用户输入query（固定token数量 = m1）
- 固定系统提示词（固定token数量 = m2） 
- 检索的知识库内容（知识库最大块token数量 * 最大检索出的块数量 = m3）
- 聊天历史（摘要历史和最近几轮完整历史 = m4 = m - m1 - m2 - m3）
### 管理聊天历史
- 摘要历史token数量 = m4 * 0.2
- 完整历史token数量 = m4 * 0.8
- 采用redis缓存管理聊天记录，提高agent的响应速度，因为聊天记录需要频繁读取，填充到上下文中。
- 当添加新的聊天记录后，完整历史token数量超过阈值，则执行摘要压缩（队列先进先出压缩旧聊天记录），将压缩后的内容放入摘要队列中，如果添加新的摘要内容后超出摘要阈值，则执行摘要删除（队列先进先出删除旧摘要内容）。
